# GPT 잡부의 세 형태

어떤 형태를 쓰는지는 두 축으로 갈린다 — **지금 세션이 프록시 base_url을 쓰는가**, 그리고 **작업이 프론트/CICD인가**.

| 형태 | 도구 | 어디서 되나 | 손은 누구 | 언제 |
|---|---|---|---|---|
| 오라클 릴레이 | 없음(추론만) | 어느 세션에서나 | Claude | 일반 세션, 프론트/CICD(게이트 필요), 교차검증·2차의견 |
| 워커 서브에이전트 | Read/Write/Edit/Bash/… | 프록시 세션 전용 | GPT 자신 | 비프론트 백엔드 잡부를 GPT가 자율 수행 |
| 메인 루프 모델 | Claude Code 전체 도구 | 프록시 세션(gpt-cc) | GPT 자신 | 실험, GPT를 주 드라이버로 |

## 1. 오라클 릴레이 (`gpt` 에이전트 / `tools/gpt-relay.ps1`)
GPT는 도구 없이 추론만. Claude가 맥락을 자기완결 프롬프트로 주고, GPT가 코드·패치를 텍스트로 반환하면 Claude가 적용·빌드·검증한다. 릴레이가 pwsh로 직접 호출하므로 **어느 세션에서나 즉시** 된다. 프론트/CICD를 GPT에 맡기지 않는 house rule과 구조적으로 정합 — Claude가 항상 게이트다. 이 토이의 두 사이클(파서, DSP)이 이 형태로 굴러갔다.

## 2. 워커 서브에이전트 (`tools/agents/gpt-worker.md`)
GPT 자체가 서브에이전트의 LLM이라 파일 읽기/편집/명령을 직접 수행한다(haiku 릴레이보다 나은 잡부). 단 `ANTHROPIC_BASE_URL`이 `gpt-proxy.mjs`를 가리키는 세션(=`gpt-cc.ps1`로 띄운 세션)에서만 작동. 일반 세션에서 부르면 `gpt-5.6-*`가 진짜 Anthropic으로 가 404. 프론트/CICD엔 부적합(자율이라 게이트가 없음) — 백엔드 로직 잡부로만.

## 3. 메인 루프 모델 (`tools/gpt-cc.ps1`)
Claude Code(앱/CLI) 전체를 GPT로 돌린다. 프록시가 Anthropic Messages API를 codex responses로 실시간 변환. 게이트가 없으니 사용자가 직접 감독.

## 고르는 법
- 일반 Claude 세션에서 큰 구현을 위임하고 싶다 → **릴레이**.
- 프론트엔드·CI/CD가 얽힌다 → **릴레이**(Claude 게이트 필수).
- GPT가 파일을 직접 만지는 자율 잡부가 필요하고, 백엔드 로직이며, 프록시 세션을 띄울 수 있다 → **워커**.
- GPT로 세션 전체를 굴려보고 싶다 → **메인 모델**.
