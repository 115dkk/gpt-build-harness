# GPT 빌드 하네스

Claude 혼자 큰 구현을 다 태우는 대신, GPT-5.6을 잡부로 붙여 구현·검증 노동을 위임하고 Claude는 설계·적용·판정을 쥐는 방식이다. GPT 호출은 ChatGPT **구독 쿼터**를 쓰므로 Claude 토큰을 아낀다.

이 문서는 개요다. 상세는 `references/`에 있다. **실행 스크립트(릴레이·프록시·런처·에이전트)는 이 저장소 소스에 없고, 릴리즈 에셋 `gpt-build-harness-tools.zip`으로 배포된다.**

이 문서의 모든 판단은 2026-07-25 실측 배터리에 근거한다. 원 데이터와 실험 설계는 `references/measurements.md`에 있다. 근거 없는 인상은 넣지 않는다.

## 언제 쓰나
- 큰 포팅·리팩터·스캐폴딩처럼 구현 노동량이 많은 작업을 Claude 토큰을 아끼며 굴리고 싶을 때.
- 한 판단을 GPT로 교차검증하거나 2차 의견을 받고 싶을 때.
- 리뷰·검증을 병렬로 분담할 때.

## 가장 중요한 것: 위임 프롬프트 규칙

모델을 고르는 것보다 프롬프트를 어떻게 쓰는가가 결과를 더 크게 가른다. 실측에서 **같은 모델·같은 effort가 프롬프트 한 문단 차이로 완전히 정직해지거나 통째로 지어냈다.** 네 규칙을 지킨다. 근거와 A/B는 `references/prompt-rules.md`.

1. **탈출구를 준다.** "없으면 `NO BUG FOUND`라고만 답하라", "확인할 수 없으면 확인할 수 없다고 하라"를 명시한다. 이 한 문단이 날조를 없앤다.
2. **산출물의 완결 조건을 못박는다.** "이 파일 하나만으로 컴파일되어야 하니 필요한 헤더와 타입 별칭을 전부 포함하라"처럼 쓴다. 모호하면 절반이 선언을 빠뜨린다.
3. **범위 밖을 미리 잘라낸다.** 보고하지 않아도 되는 것(불변식, 형식론적 UB, 스타일)을 계약으로 명시한다. 그러면 소견이 진짜 결함에 집중된다.
4. **검증할 수 없는 것을 요구하지 않는다.** 릴레이 GPT는 저장소를 못 본다. 저장소 특정 사실은 프롬프트 본문에 붙여 주고, 실행 여부는 `VERIFIED: compiled=<yes|no> tested=<yes|no>` 같은 신고 줄로 물어 확인한다.

명세를 얼마나 주어야 하는지는 따로 다룬다. 명세에는 층위가 있어 층마다 막는 실패가 다르고, 미감 회귀(slop)를 막는 데 가장 남는 투자는 픽셀 표가 아니라 철학 한 장이다. 계단식 실험은 `references/delegation-boundary.md`.

## GPT 잡부의 세 형태
어느 형태를 쓰는지는 **지금 세션이 프록시 base_url을 쓰는가**와 **GPT가 도구를 직접 쥐어도 되는가**로 갈린다. 판단표는 `references/three-forms.md`.

1. **오라클 릴레이 (`gpt` 에이전트)**: GPT는 도구 없이 추론만 한다. Claude가 맥락을 프롬프트에 담아 주고, GPT가 코드·소견을 텍스트로 반환하면 Claude가 적용·빌드·판정한다. **어느 세션에서나 즉시** 된다. Claude 게이트가 구조적으로 강제된다.
2. **워커 서브에이전트 (`gpt-worker` 에이전트)**: GPT 자체가 서브에이전트의 LLM이라 파일 읽기·편집·명령을 직접 한다. `ANTHROPIC_BASE_URL`이 프록시를 가리키는 세션에서만 작동한다.
3. **메인 루프 모델 (런처)**: Claude Code 전체를 GPT로 돌린다.

**탐색은 형태가 결정한다.** 릴레이 GPT는 도구가 없어 탐색을 아예 못 한다. 일반 세션에서 코드베이스 탐색은 Claude(Explore)가 맡는다. GPT에 탐색을 시키려면 프록시 세션이어야 하고, 프록시의 도구 호출 변환은 실측으로 확인했다.

## 모델·effort 고르기 (실측 요약)

정확한 수치와 실험 설계는 `references/tiers.md`.

| 역할 | 권장 | 근거 요약 |
|---|---|---|
| 구현(스펙이 정밀할 때) | **luna low** | 30초·1,625토큰에 숨은 테스트 35/35, `/W4` 경고 0 |
| 리뷰·검증(기본) | **sol medium** | 의미·순서 결함을 2회 모두 22~39초에 적중 |
| 리뷰·검증(더 싸게) | terra medium | 13초·675토큰 적중, 깨끗한 코드엔 오탐 0 |
| 교차검증·2차 의견·거짓 전제 위험 | **sol high** | 탈출구 없이도 거짓 전제를 스스로 거부한 유일한 모델 |

`GPT-EFFORT`를 습관적으로 `xhigh`로 두지 않는다. 실측에서 luna xhigh는 luna medium과 같은 소견을 내면서 시간과 토큰을 9배 썼다. **effort는 단조가 아니다.** 호출할 때 effort를 항상 명시한다.

Luna max는 사용자 평가가 좋아 계속 쓴다. 다만 이번 검증 배터리에서 2회 중 1회 놓쳤으니, 판정 하나를 max 단독에 맡기지 않는다. SOL은 high를 넘기지 않는다(고정 방침).

## GPT에 맡기지 말 것
- **프론트엔드 디자인 결정**: GPT 감각이 나쁘다. 확정된 디자인의 렌더 번역은 위임해도 된다(`references/orchestration.md`).
- **CI/CD 파이프라인 설계**: 자주 깬다.
- 메인·워커 형태는 Claude 게이트가 없으니 사용자가 직접 감독한다.

## 검증된 함정
`references/pitfalls.md`에 근거와 함께 모았다. 특히 자주 밟는 것.

- 릴레이는 pwsh 7로만 실행한다(WinPS 5.1은 네트워크가 막힌다).
- **릴레이 출력은 PowerShell `>` 리다이렉션으로 못 잡는다**(`[Console]::Out.Write`가 콘솔 핸들에 직접 쓴다).
- **에이전트·Workflow 결과의 HTML 엔티티는 표시 계층 현상이다.** 손으로 복원하지 말고 `journal.jsonl`에서 원문을 꺼낸다. Claude 에이전트 결과도 똑같이 이스케이프된다.
- 프록시는 codex responses에 `max_output_tokens`를 보내면 안 된다(400).
- 워커·메인 형태는 프록시 세션 전용이다. 일반 세션에서 `gpt-5.6-*`를 부르면 404.
