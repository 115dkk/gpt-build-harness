# GPT 빌드 하네스

Claude 혼자 큰 구현을 다 태우는 대신, GPT-5.6을 잡부로 붙여 구현 노동을 위임하고 Claude는 설계·적용·검증만 쥐는 방식이다. GPT 호출은 ChatGPT **구독 쿼터**를 쓰므로 Claude 토큰을 아낀다.

이 문서는 개요다. 상세는 `references/`로 분산한다. **실행 스크립트(릴레이·프록시·런처·에이전트)는 이 저장소 소스에 없고, 릴리즈 에셋 `gpt-build-harness-tools.zip`으로 배포된다** — README의 설치 안내를 따르면 Claude Code / Codex / Claude 앱에 배치된다. 이 스킬은 Qt 데스크톱 앱을 Tauri로 포팅하는 토이를 굴리며 벼려졌으며, 그 교훈은 `references/harness-log.md`에 시간순으로 기록되어 있다.

## 언제 쓰나
- 큰 포팅/리팩터/스캐폴딩처럼 구현 노동량이 많은 작업을 Claude 토큰을 아끼며 굴리고 싶을 때.
- 한 판단을 GPT로 교차검증하거나 2차 의견을 받고 싶을 때(판정 패널의 이견 멤버).
- 지식 조회, 백엔드 설계/구현 검토를 GPT에 분담할 때.

## GPT 잡부의 세 형태
어떤 형태를 쓰는지는 **현재 세션이 프록시 base_url을 쓰는지**와 **작업이 프론트/CICD인지**로 갈린다. 자세한 판단표는 `references/three-forms.md`.

1. **오라클 릴레이 (`gpt` 에이전트)**: GPT는 도구 없이 추론만 수행한다. Claude가 맥락을 프롬프트에 담아 주고, GPT가 코드·패치를 텍스트로 반환하면 Claude가 적용·빌드·검증한다. **어느 세션에서나 즉시** 된다. 프론트/CICD를 GPT에 맡기지 않는 house rule과 구조적으로 정합되어 Claude가 항상 게이트 역할을 한다.
2. **워커 서브에이전트 (`gpt-worker` 에이전트)**: GPT 자체가 서브에이전트의 LLM이라 파일 읽기/편집/명령을 **직접** 수행한다. 단 `ANTHROPIC_BASE_URL`이 프록시를 가리키는 세션에서만 작동한다. 일반 세션에서 부르면 `gpt-5.6-*`가 진짜 Anthropic으로 가 404 오류가 난다.
3. **메인 루프 모델 (런처)**: Claude Code(앱/CLI) 전체를 GPT로 돌린다. 프록시가 Anthropic Messages API를 codex responses로 실시간 변환한다.

## 셋업 (릴리즈 설치)
실행 도구는 릴리즈 zip에 들어 있다. 설치하면 릴레이(구독 오라클), 프록시(Anthropic↔codex 변환), 런처(claude 기동), 서브에이전트 정의(`gpt` / `gpt-worker`)가 배치된다. 인증은 로컬 ChatGPT 구독 토큰을 쓴다(API 키 아님). 개요는 `references/setup.md`, 구체 프로토콜은 설치된 도구 주석에 있다.

## 오케스트레이션 루프 (릴레이 형태의 정석)
1. **Claude가 설계·범위를 확정한다**. 특히 프론트엔드는 Claude가 먼저 확정해야 한다. 무엇을 만들지, 인터페이스가 어떤지 GPT에 맡기지 않는다.
2. **맥락을 자기완결 프롬프트로 GPT에 위임한다**. GPT는 대화 맥락·파일 접근이 없으므로, 필요한 코드/스펙/기존 파일 내용을 프롬프트 본문에 붙여 준다.
3. **GPT가 반환한 코드를 Claude가 적용하고 빌드·테스트한다**.
4. **실패하면 에러 로그를 다시 GPT에 붙여 재질의한다**. 성공까지 반복한다.
5. 겪은 교훈을 기록한다.

선명세가 재질의를 줄인다 — 인터페이스(타입/시그니처/테스트 기준)를 Claude가 못박을수록 재질의 횟수가 줄어든다. 자세히는 `references/orchestration.md`.

## GPT에 맡기지 말 것 (house rule)
- **프론트엔드 디자인**: GPT 감각이 나쁘다. 설계는 Claude가 확정하고, 구현·검토만 위임한다.
- **CI/CD 파이프라인**: GPT가 자주 깬다. 품질을 먼저 확인하기 전에는 맡기지 않는다.
- 메인 모델·워커 형태일 때는 Claude 게이트가 없으므로 사용자가 직접 감독한다.

## 티어 운용
- Luna xhigh/max = 다역 잡부(기본). Luna low~medium = 아주 단순한 작업만.
- SOL medium = 지식 집약 잡부, SOL high = 교차검증·백엔드 구현 검토(SOL은 high 초과 금지).
- Terra 비권장. 메인/워커로 매 턴 고effort는 느리니 기본 medium.

## 검증된 함정
`references/pitfalls.md`에 근거와 함께 모았다. 요약:
- 릴레이는 pwsh 7로만 실행(WinPS 5.1은 네트워크가 막힌다).
- 프록시는 codex responses에 `max_output_tokens`를 보내면 안 됨(400 오류).
- codex 자동 캐시는 session_id 헤더에 묶임 — 대화별 안정 session_id를 써야 캐시가 걸린다.
- 워커/메인 형태는 프록시 세션 전용. 일반 세션에서 `gpt-5.6-*`를 부르면 404 오류.
- 릴레이 반환 코드가 에이전트 결과로 올 때 `<` `>` `&`가 HTML 엔티티로 이스케이프된다 — 심기 전 복원해야 한다.

## 어디서 왔나
이 스킬은 Qt 데스크톱 앱의 한 조각을 Tauri 웹뷰로 포팅하는 토이(비출시)를 굴리며 벼려졌다. 형성 과정은 `references/harness-log.md`에 남아 있다.
